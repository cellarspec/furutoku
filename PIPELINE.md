# PIPELINE.md — 投稿コンテンツ生成手順書(定期生成エージェント向け)

このリポジトリは「ふるとく研究室」(主婦向け・ふるさと納税の比較/制度解説 Instagramアカウント)の
カルーセル投稿を全自動生成・全自動投稿する。コンテンツ生成エージェントは本書に**厳密に**従うこと。

## 全体フロー

```
[このエージェント] content/queue/*.json を生成して main に push
        ↓ (毎日 12:00 JST, GitHub Actions)
check-content.mjs(品質ゲート)→ render.mjs(JPEG化)→ 画像push → post-instagram.mjs(投稿)
        ↓
投稿済み JSON は content/posted/ へ自動移動
```

- 生成エージェントの仕事は **queue に JSON を積むところまで**。画像化・投稿には関与しない。
- 生成後は必ずローカルで `npm run check` 相当の検証が通る内容にすること(ゲートNGだと当日の投稿が止まる)。

## 投稿頻度と型のローテーション

- 通常期: **週3〜4本**。10〜12月(駆け込み需要期)は**週5本**。queueには常に3〜7本を維持する。
- 投稿の型は以下をローテーションし、同じ型を2連続させない:
  1. **カテゴリ別 還元率TOP5**(米/肉/海鮮/フルーツ/日用品 など)
  2. **寄付額帯別ベスト返礼品**(1万円/2万円/3万円帯)
  3. **量コスパランキング**(内容量÷寄付額)
  4. **控除上限の早見表**(年収×家族構成)
  5. **制度解説**(ワンストップ特例/確定申告/よくある失敗)
  6. **季節もの**(締切カレンダー、旬の返礼品、年末駆け込み対策)

## 裏取りルール(最重要)

- **数値・制度・商品情報は一次ソースで確認できたものだけ書く**。確認できない値は投稿に入れない。
  - 制度: 総務省ふるさと納税ポータル https://www.soumu.go.jp/main_sosiki/jichi_zeisei/czaisei/czaisei_seido/furusato/
  - 返礼品(寄付額・内容量): 楽天ふるさと納税/さとふる/ふるなび の**該当返礼品ページ**
  - 還元率 = 市場価格 ÷ 寄付額 を**自社算出**する。市場価格は通常販売の同等品ページ(楽天市場・Amazon等)で確認し、算出根拠のURLを `sources` に残す。**他社の還元率ランキングの転載は禁止**(数字だけの引用も不可)。
- `sources` には使用した一次ソースURLを全て入れる(1件以上必須、ゲートで検証される)。
- 返礼品情報は変動が激しい。queue投入から投稿まで日が空くため、**寄付額や内容量が変わりうる旨の注記**(「2026年7月時点」等)をcaptionに入れる。

## NGルール(品質ゲートで機械検証される)

1. **断定的な利益表示の禁止**: 「必ず得」「絶対お得」「確実に儲かる」「元本保証」「損しません」等。
2. **ポイント訴求の禁止**: 2025年10月からポータルサイト経由の寄付者向けポイント付与は制度上禁止された。
   「ポイント還元」「ポイント付与」等の訴求は誤情報になるため書かない。
3. **PR表記**: アフィリエイトリンクへ誘導する投稿は `affiliate: true` にし、captionの**冒頭**に
   「【PR】この投稿はアフィリエイト広告を含みます」を入れる(ステマ規制対応。ゲートで検証)。
4. **スライド本文に絵文字禁止**(レンダラが描画できない)。captionには使用可。
5. 金融商品(株・投資信託・保険等)の推奨に踏み込まない。ふるさと納税と税控除の範囲に留める。

## queue JSON スキーマ

ファイル名: `content/queue/YYYY-MM-DD-<slug>.json`(日付は投稿希望日。ファイル名順に1日1件消化される)

```jsonc
{
  "slug": "onestop-tokurei-5steps",   // 英小文字・数字・ハイフン。画像出力先になる
  "title": "管理用タイトル",
  "caption": "Instagram本文(ハッシュタグは含めない)",
  "hashtags": ["#ふるさと納税", "..."], // 25個まで
  "affiliate": false,                  // アフィ誘導ならtrue+captionにPR表記
  "sources": ["https://..."],          // 裏取りに使った一次ソースURL(必須)
  "slides": [                          // 2〜10枚。1枚目=cover、最終=cta 固定
    { "type": "cover", "kicker": "帯文言(〜20字)", "title": "見出し(1行13字まで・\\nで改行)", "sub": "補足(〜44字)" },
    { "type": "list",  "heading": "見出し(〜24字)", "items": [{ "title": "〜30字", "body": "〜64字" }] },  // itemsは4件まで
    { "type": "steps", "heading": "見出し", "items": [{ "title": "手順名", "body": "説明" }] },  // 番号は複数スライドで通し番号
    { "type": "note",  "heading": "見出し", "body": "強調文(\\n改行可・計90字まで)", "footnote": "注記(〜80字)" },
    { "type": "cta",   "title": "締め(\\n改行可・計30字)", "body": "プロフィールリンク誘導文(〜90字)" }
  ]
}
```

- 文字数上限はレイアウト崩れ防止のため。超えるとゲートNG。
- ランキング系は `list` を使い、`title` に「1位 〇〇(△△市)」、`body` に「寄付額◯円/内容量◯kg/根拠」を書く。

## 生成のトーン

- ターゲット: 30〜40代主婦。「難しそう」「損したくない」に寄り添う。専門用語は言い換える。
- coverは数字や意外性のあるフックを使う(「知らないと損」「1行で言うと」等)。誇張はNGルールの範囲内で。
- captionは絵文字を適度に使い、最後に「※制度の詳細は総務省ふるさと納税ポータルをご確認ください」を入れる。

## 月次メンテナンス(人間または月次エージェント)

- 掲載済み返礼品の寄付額・取扱終了を月1で再確認し、大きく変わっていたら該当postedを参照する新規投稿で訂正。
- 年次(12月末〜1月): 制度改正・NISA等ではなく**ふるさと納税の税制変更**を総務省ポータルで確認し、PIPELINE.mdの数値を更新。
